# Vercel Deploy (saas skill)

> Production-grade Vercel deployment for Next.js SaaS. Default deploy target of `saas/b2c-subscription`.

## Role

You get a Next.js SaaS deployed to Vercel with custom domain, HTTPS, proper env var separation, preview deployments per PR, and basic monitoring.

## 2026 Best Practices

- **Production / Preview / Development envs separated** — never share secrets across.
- **All secrets in Vercel env, never committed** — `.env.local` is local-only.
- **Custom domain in Vercel Dashboard** — DNS via Cloudflare or your provider; HTTPS auto.
- **Preview deployment per PR** — auto-generated by GitHub integration.
- **ISR for marketing pages** — `revalidate: 60` for blog/docs that aren't truly dynamic.
- **Edge Runtime for middleware + auth-heavy routes** — geographically close = lower latency.
- **Vercel Analytics for Core Web Vitals** — free tier sufficient for early SaaS.
- **Function timeouts set explicitly** — default 10s on hobby, 60s on pro; declare per route.

## Implementation pattern

### 1. Connect GitHub repo + create project

```
1. Sign in to Vercel
2. Click "Add New" → "Project"
3. Connect GitHub, select repo
4. Framework preset: Next.js (auto-detected)
5. Build & Output Settings: leave defaults
```

### 2. Environment variables (Vercel Dashboard)

Add per-environment:
```
Production:
  DATABASE_URL=postgres://...prod
  CLERK_SECRET_KEY=sk_live_...
  STRIPE_SECRET_KEY=sk_live_...
  STRIPE_WEBHOOK_SECRET=whsec_live_...
  NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_live_...
  NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
  RESEND_API_KEY=re_live_...
  SENTRY_DSN=...
  SENTRY_AUTH_TOKEN=...

Preview:
  DATABASE_URL=postgres://...staging
  CLERK_SECRET_KEY=sk_test_...
  STRIPE_SECRET_KEY=sk_test_...
  STRIPE_WEBHOOK_SECRET=whsec_test_...
  # ... test/preview versions ...
```

Never use prod Stripe in preview — bad card events go to live.

### 3. Custom domain

```
1. Vercel Project → Settings → Domains
2. Add: yourapp.com
3. Vercel shows DNS records: ALIAS or A
4. Add to DNS provider (Cloudflare proxy OFF on Vercel ALIAS records)
5. Wait ~5 min for verification
6. HTTPS auto-provisioned via Let's Encrypt
```

### 4. Edge vs Node runtime

```typescript
// app/api/some-route/route.ts
export const runtime = 'edge';     // global, fast, limited APIs
// OR
export const runtime = 'nodejs';   // default, full Node APIs
```

Use edge for: middleware, auth checks, geo-aware redirects.
Use node for: Stripe webhooks, heavy DB queries, anything > 1s.

### 5. Function timeouts

```typescript
// app/api/process/route.ts
export const maxDuration = 60;   // seconds (Pro plan = 60s max, Hobby = 10s)
```

### 6. ISR for content pages

```typescript
// app/blog/[slug]/page.tsx
export const revalidate = 3600;   // 1 hour

export default async function Post({ params }) {
  const post = await fetchPost(params.slug);
  return <article>{...}</article>;
}
```

### 7. Vercel Analytics (Core Web Vitals)

```bash
npm install @vercel/analytics
```

```tsx
// app/layout.tsx
import { Analytics } from '@vercel/analytics/react';
<body>
  {children}
  <Analytics />
</body>
```

### 8. GitHub integration

Once connected, every PR auto-deploys to a preview URL like `yourapp-pr-123.vercel.app`. The Vercel bot comments on the PR with the URL. Use it for QA before merge.

## Critical pitfalls

1. **Prod Stripe keys in preview env** — preview deployments charge real cards. Always use `sk_test_` in preview.
2. **No custom domain** — `yourapp.vercel.app` looks unprofessional; emails go to spam (DKIM/SPF on Vercel subdomain is harder).
3. **Edge runtime + Node-only library** — fails at build time. Check library compatibility (most ORMs are Node-only).
4. **Default 10s timeout on long-running route** — Stripe portal callback can be slow; set `maxDuration: 30`.
5. **HSTS not set on subdomain** — Vercel only auto-HSTS on the apex domain. Subdomains need explicit headers.
6. **No `.vercelignore`** — `node_modules` ends up in builds. Add `.vercelignore` mirroring `.gitignore`.

## Verification post-deploy

```bash
# HTTPS works + redirects from HTTP
curl -sI http://yourapp.com | grep -E "(HTTP/|Location)"

# Custom domain resolves
curl -sI https://yourapp.com | head -5

# Function works
curl -s https://yourapp.com/api/health

# Vercel Analytics receiving (check dashboard)
```
